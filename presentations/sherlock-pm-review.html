Sherlock — my personal research analyst.

A two-tier LLM system that runs my content diet. Quick lookups in iMessage with citations; deep, multi-source memos written straight to my Obsidian vault. Built to maximise the signal-to-noise ratio of what I read every day.

The thesis
I needed an analyst, not a feed reader.
Two modes · one identity · zero hallucinated citations.
01 · INGEST
Curates my corpus 24/7
Four cron'd cloud agents pull YouTube transcripts, Substack, X posts, and blog RSS into a versioned Markdown corpus. ≤1 hour after publish.
02 · QUICK
Analyst replies in ~16s
Front triangulates vault + web + thread, replies in ≤4 sentences with structured citations, refuses deep synthesis itself.
03 · DEEP
McKinsey-style memo to Obsidian
5–15 min sub-agent. Claim-first sections, pyramid principle, structured citations, mandatory Confidence block.
04 · ONBOARD
Add a source from a pasted URL
"add this: <url>" → auto-detects YouTube / X / Substack / RSS, validates, dedupes, commits to the source roster.
05 · NOTIFY
Pings me only when there's something to read
No progress spam. One iMessage per finished report with the obsidian:// deep link and a 3-sentence TL;DR.
06 · LOCAL
Runs on my Mac, never leaks
Bridge + indexer + repo sync as 4 launchd jobs. Cloud only writes a separate, write-isolated repo. My notes never leave my machine.

The whiteboard view

Three repos · two-tier agents · MCP as the contract.

routed via @cursor/sdk + Anthropic SDK · launchd × 4
Decision · 01
Two-tier agents
Fast Front (Haiku) handles every turn. Slow Researcher (Sonnet) handles synthesis. Async lifecycle keeps iMessage responsive.
Decision · 02
Direct Anthropic SDK for Front
Cursor SDK injects "you are a coding assistant" the model treats as authoritative. Traded SDK convenience for persona stickiness.
Decision · 03
Mandatory Y/N gate
One explicit human checkpoint before any agent burns 10 min and a deep_research call. Literal Y or yes — not soft confirmation.
Decision · 04
Git as the database
Three repos = three writers, zero contention. Free durability, audit trail, sync. Cloud writes corpus; local writes vault.

How the agents coordinate

MCP is the contract; the queue is the trust boundary.

  • Tool surface = trust boundary. Front gets read + control tools (no report-writer, no parallel-task). Researcher gets write + heavy-web (no research-control). No agent can spawn another.
  • Async via durable queue. research.start returns instantly with { id, queue_pos, eta }. Job manager owns a SQLite-backed FIFO with cap=3 + 30-min hard timeout. Survives bridge restart.
  • No cross-agent shared state — every handoff is a tool call with a typed payload, or a git commit. Two different Claude families, one analyst identity.
  • No partial silence. Failed researchers still finalize a status: partial report and notify me. The system never goes dark.

Future improvements

The next six bets — ranked by user-felt impact.

BET 01 · UX
Daily Brief sub-agent
A 6 a.m. proactive Researcher that surfaces the three things from yesterday's corpus I should actually read. Inverts "I ask, it answers."
BET 02 · QUALITY
Self-criticism revision pass
Second Sonnet pass grades the draft against the McKinsey contract and forces a revision before notify_complete fires.
BET 03 · RETRIEVAL
Personal entity graph
Extract people, companies, topics on ingest. "Everything Dwarkesh said about scaling laws this year" becomes a graph hop, not a search.
BET 04 · CORPUS
Multi-modal ingest
Whisper for podcasts. PDFs dropped into a vault inbox auto-indexed. X bookmarks once user-context OAuth lands.
BET 05 · COST
Tiered model router
Trivia → Haiku, recon → Sonnet, deep synthesis → Opus. Promote the implicit prompt-level routing into an explicit policy module.
BET 06 · LOOP
Source-quality scoring
Track which sources Front actually cites in real reports. Auto-suggest dropping sources with no citation in 90 days. Corpus curates itself.
